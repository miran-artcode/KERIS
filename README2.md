# 연수 반응 로그 시스템

음악·미술·체육 창작교육 데이터 강의(11차시)에서 연수생 각자의 참여·반응을 실시간으로 기록하고,
그 로그를 12차시 환류 실습의 표본 데이터로 넘기기 위한 배포형 웹 시스템입니다.

## 구성 파일

- `instructor.html` — 강의자 콘솔. 세션 생성, 슬라이드 표시, 단계 표시, 발화 전사, 즉석 폴, 실시간 참여 현황, 내보내기.
- `participant.html` — 연수생 접속 화면. 동의 후 반응 버튼, 메모, 폴 응답, 웹캠 주의 신호(선택), 음성 녹음(선택).
- `config.js` — Supabase 연결 값과 캡처 기본값.
- `supabase_schema.sql` — 데이터베이스 테이블, 실시간 활성화, 접근 정책.

## 데이터 흐름

연수생 기기(participant.html)가 이벤트를 Supabase에 기록하고, 강의자 콘솔(instructor.html)이
같은 세션의 이벤트를 실시간 구독해 현황판에 표시합니다. 기록되는 이벤트 종류는 접속, 단계 전환,
폴 응답, 반응 버튼, 메모, 웹캠 주의 신호, 스냅샷, 음성, 하트비트입니다. 강의자 발화는 별도 전사 테이블에 쌓입니다.

## 준비 순서

1. Supabase에서 새 프로젝트를 만듭니다(무료 요금제로 시작 가능).
2. 대시보드의 SQL Editor에 `supabase_schema.sql` 내용을 붙여넣고 실행합니다.
3. Storage에서 `snapshots` 버킷을 만들고 비공개로 둡니다(음성·스냅샷 저장용).
4. Project Settings → API에서 URL과 anon key를 복사해 `config.js`에 넣습니다.
5. 네 파일을 같은 폴더에 두고 정적 호스팅에 올립니다(Netlify, Vercel, GitHub Pages 등).
6. 강의자는 `instructor.html`, 연수생은 `participant.html`로 접속합니다.

## 실행 조건

카메라와 마이크는 HTTPS 또는 localhost에서만 열립니다. 정적 호스팅은 대부분 HTTPS를 제공하므로 배포 후에는 문제없습니다.
실시간 전사는 Web Speech API를 쓰므로 Chrome 계열에서 동작합니다.

## 웹캠 주의 신호의 성격

웹캠 항목을 켜면 얼굴 인식(MediaPipe FaceLandmarker)으로 계산한 요약 수치만 기록됩니다.
화면 응시 추정과 눈 뜸 정도를 합쳐 0~1 사이 점수를 3초 간격으로 남기며, 원본 영상은 저장하지 않습니다.
이 점수는 근사값이라 주의의 방향을 대략 보여 주는 신호로만 쓰고, 개인 평가의 근거로 삼지 않도록 안내하십시오.
정밀 시선 좌표가 필요하면 WebGazer.js로 보정 단계를 추가해야 하며, 그래도 웹캠 기반 정확도는 제한적입니다.

## 동의·개인정보 처리

- 연수생 화면은 접속 전에 항목별 동의 체크(음성, 웹캠 주의 신호, 스냅샷)를 받고, 동의하지 않은 항목은 수집하지 않습니다.
- 스냅샷은 가장 민감한 항목이라 기본이 꺼짐이며, 켠 경우에만 1분 간격 축소 이미지가 저장됩니다.
- 보관 기간 안내 문구는 `config.js`의 `RETENTION_NOTICE`에서 바꿉니다. 연수 종료 후에는 스키마 파일 하단의 삭제 SQL로 데이터를 지우십시오.
- anon key는 브라우저에 노출됩니다. 접속 코드가 추측되지 않게 관리하고, 더 엄격한 통제가 필요하면 Supabase Auth를 붙여 정책을 사용자 기준으로 바꾸십시오.
- 이 연수가 학습 데이터 보안·개인정보보호(③차시)를 가르치므로, 동의 화면과 보관 정책을 강의 예시로 함께 보여 주면 내용과 운영이 일치합니다.

## 12차시 환류로 넘기기

강의자 콘솔의 JSON 내보내기는 이벤트·전사·참가자를 한 파일로 묶습니다. 이벤트 CSV는 시간·참가자·유형·내용을 표로 폅니다.
12차시에서는 이 로그를 세 축(강의자·연수생 발화 비율, 개방형 질문 응답 분포, 지원 발문·막힘 반응)으로 분석하고,
과목 분반에서 각자 자기 반응 로그를 해석해 개선 지점 하나를 도출하는 실습으로 연결합니다.
발화 비율이나 응답 수는 그 회차의 기술로만 제시하고, 여러 수업에 걸친 일반 경향의 근거로는 쓰지 않습니다.

## 지금 스캐폴드의 한계

- 접근 정책이 워크숍용 최소 수준입니다. 공개 행사에 그대로 쓰기 전에 Auth와 정책 강화를 검토하십시오.
- 음성 파일과 스냅샷은 Storage 용량을 빠르게 씁니다. 인원과 시간에 맞춰 주기와 보관 기간을 조정하십시오.
- 웹캠 주의 신호는 조명·자세·카메라 각도에 영향을 받습니다.
